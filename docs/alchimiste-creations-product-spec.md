# Alchimiste Créations

# Vision produit

# 1. Présentation du projet

**Alchimiste Créations** est un studio web indépendant spécialisé dans la création de sites web modernes et performants.

Le studio accompagne les entrepreneurs, créateurs et indépendants qui souhaitent présenter leur activité de manière claire, professionnelle et crédible sur internet.

**Slogan**

> Transformer les idées et les projets en sites web modernes et crédibles.

L’objectif est de transformer une idée, une activité ou un projet en un site web qui inspire confiance et permet de toucher les bonnes personnes.

---

# 2. Mission

La mission d’Alchimiste Créations est d’aider les entrepreneurs et créateurs à transformer leur projet en un site web clair, moderne et crédible.

Le studio accompagne les projets porteurs de sens en créant des sites web qui permettent de :

- présenter une activité de manière professionnelle
- transmettre une valeur ou un message
- inspirer confiance
- attirer les bons clients

# 3. Objectif principal du site

# 6. Rôle des dossiers

## `app/`

Le dossier `app/` contient :

- les routes
- les layouts
- la composition des pages

Il ne doit pas contenir toute la logique métier.

Son rôle est de gérer le **routing** et l’**assemblage**.

Les pages publiques du site sont regroupées dans `(marketing)`, y compris les pages légales, afin de partager le même layout public.

## `components/`

Le dossier `components/` contient les composants React.

### `ui/`

Composants génériques et réutilisables dans tout le projet.

Exemples :

- Button
- Card
- Input
- Textarea
- Heading
- Container

### `layout/`

Composants structurels globaux.

Exemples :

- Header
- Footer
- Navigation
- Section

### `marketing/`

Sections utilisées pour les pages marketing.

Exemples :

- Hero
- CTA
- ServicesPreview
- PortfolioPreview
- Process

### `services/`, `portfolio/`, `contact/`

Composants métiers spécifiques à un domaine du site.

Cette organisation permet d’éviter un dossier `components/` désordonné.

## `content/`

Le dossier `content/` centralise les contenus du site :

- textes marketing
- données des offres
- contenus des pages
- contenus du portfolio

L’objectif est de **séparer le contenu de l’interface**.

Avantages :

- contenu plus facile à modifier
- JSX plus lisible
- meilleure maintenabilité
- possibilité future de brancher un CMS

## `domain/`

Le dossier `domain/` contient les types métier du projet.

Exemples :

- `Project`
- `Service`
- `ContactRequest`

Il sert à poser les bases métier du site de manière claire.

## `application/`

Le dossier `application/` contient les cas d’usage.

Exemples :

- récupérer les projets mis en avant
- récupérer tous les services
- récupérer la navigation
- récupérer un projet par slug

Cette couche permet de séparer la logique métier du rendu UI.

## `infrastructure/`

Le dossier `infrastructure/` regroupe les éléments techniques dépendants d’outils ou de services externes.

Exemples :

- envoi d’email
- analytics
- accès aux données portfolio

C’est ici qu’on place ce qui pourra évoluer selon les outils utilisés.

## `lib/`

Le dossier `lib/` contient les briques transverses :

- utilitaires
- constantes
- validateurs
- gestion des variables d’environnement

## `styles/`

Le dossier `styles/` contient les styles globaux et les fondations du design system.

Exemples :

- design tokens
- utilitaires CSS
- styles globaux de composants

Le fichier `app/globals.css` sert de **point d’entrée global** chargé par Next.js et peut importer les fichiers du dossier `styles/`.

## `types/`

Le dossier `types/` contient les types globaux non directement liés au métier.

Exemples :

- types API partagés
- types techniques communs
- types utilitaires globaux

## `tests/`

Le dossier `tests/` prépare une base de tests propre dès le départ.

Il pourra contenir :

- tests unitaires
- tests d’intégration
- fixtures de données
